# v0.8.55 Module Boundary Enforcement and Project Port Migration

This iteration tightens the modular architecture introduced in v0.8.53 and v0.8.54.

## Completed

- Project Port contracts now expose a stable read/write boundary:
  - `ProjectReadPort`
  - `ProjectWritePort`
  - `ProjectContext`
  - `ProjectResourceSummary`
  - `project_document_from()`
- Module facades accept Project Ports directly:
  - `modules.meshing.generate_project_mesh()`
  - `modules.stage_planning.compile_project_stages()`
  - `modules.fem_solver.solve_project()`
  - `modules.postprocessing.summarize_results()`
- Plugin descriptors are normalised around:
  - `PluginCapability`
  - `PluginHealth`
  - `PluginDependencyStatus`
- Example replacement plugins verify registry interoperability without GUI/service changes:
  - `DummyMeshGenerator`
  - `DummySolverBackend`
  - `DummyPostProcessor`
- GUI-facing orchestration has new split points:
  - `app.controllers`
  - `app.view_models`
  - `app.presenters`
- Architecture tests now enforce service/module import direction, Project Port use and plugin descriptor shape.
